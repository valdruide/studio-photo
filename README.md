## üóÑÔ∏è Data Architecture (PocketBase)

#### categories

| Parameter | Type     | Description                |
| :-------- | :------- | :------------------------- |
| `slug` | `string` | **Required**. Slug of the category |
| `title` | `string` | **Required**. Title of the category |
| `order` | `number` | **Required**. Position of the category |
| `isHidden` | `boolean` | **Required**. If the category is hidden  |
| `color` | `string` | Color of the category title  |
| `icon` | `string` | Icon before the category title - based on [React Tabler icons](https://tabler.io/icons) |

#### photo_collections

| Parameter | Type     | Description                |
| :-------- | :------- | :------------------------- |
| `slug` | `string` | **Required**. Slug of the collection |
| `title` | `string` | **Required**. Title of the collection |
| `description` | `string / rich` | Description of the collection (rich text) |
| `category` | `category relation` | **Required & single**. Category related to the collection |
| `order` | `number` | **Required**. Position of the collection |
| `isHidden` | `boolean` | **Required**. If the collection is hidden  |

#### photos

| Parameter | Type     | Description                |
| :-------- | :------- | :------------------------- |
| `collection` | `collection relation` | **Required & single**. Collection related to the photo |
| `name` | `string` | **Required**. Name of the photo |
| `description` | `string / rich` | Description of the photo (rich text) |
| `image` | `file` | **Required & single**. Image (Define the thumbnail sizes EXACTLY the same as in `/lib/pb/thumbs.ts`) |
| `order` | `number` | **Required**. Position of the photo |
| `isHidden` | `boolean` | **Required**. If the photo is hidden  |
| `width` | `number` | **Required**. Image size automatically generated by the server. No need to fill this field manually  |


## ‚öôÔ∏è Scripts & Automation
### `/scripts/import-pocketbase.mjs` - with Nodejs

**Bulk Photo Import Script**

The project includes a bulk import script designed to populate the database with categories, collections, and photos in a single pass.

This script is intended for:
- Initial project setup
- Large photo libraries
- Fast migrations or re-imports

**It handles:**
- Categories ‚Üí collections ‚Üí photos hierarchy
- File uploads
- Ordering
- Visibility flags
- Optional HTML descriptions
- Duplicate prevention

**‚ñ∂Ô∏è Run the import script** `node scripts/import-pocketbase.mjs`

> ‚ö†Ô∏è **The script expects a JSON file** named `import-data.json` describing the full content tree and local paths to image files.

### `import-data.json` structure (Template)
```json
{
    "categories": [
        {
            "slug": "category-slug",
            "title": "Category title",
            "order": 1,
            "isHidden": false,
            "collections": [
                {
                    "slug": "collection-slug",
                    "title": "[ Collection Title ]",
                    "descriptionHtml": "<p>Optional HTML description</p>",
                    "order": 1,
                    "isHidden": false,
                    "photos": [
                        {
                            "file": "public/series/Collection/image-1.png",
                            "name": "Photo title",
                            "descriptionHtml": "<p>Optional HTML description</p>",
                            "order": 1,
                            "isHidden": false
                        }
                    ]
                }
            ]
        }
    ]
}
```
**Notes**
- `file` must point to a local image path
- `descriptionHtml` fields are optional
- Ordering is explicit and respected during import
- All visibility is controlled via `isHidden`

This script is designed as a reusable building block, with potential evolution toward a more advanced admin dashboard.
